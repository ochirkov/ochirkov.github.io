Continuous delivery for Jenkins Jobs...

Posted by Oleksandr Chyrkov on December 11, 2017

Jenkins is a universal tool that can be used for different purposes, be it continuous delivery of software, different scenarios execution, running tests, or deployment of infrastructure. This product has many practical applications.

As the number of configurations and their complexity keeps increasing, build engineers, developers and system administrators have to invest more time in their maintenance, development and testing. Team work on configurations via WEB UI can be challenging enough, and testing even more so.

There are a few tools that make real configurations development in a team, versioning of changes, and flexible testing in real time.

Ease of use and flexible approach make Jenkins Job Builder a perfect solution for big projects on Jenkins. Jenkins native Pipeline allows simply build your pipelines with DSL blocks.

This cookbook attempts to compile all possibilities for above technologies and practical examples of how you can use them. You can download few chapters of "Jenkins. Configuration as code." and content of book by clicking the link below.
